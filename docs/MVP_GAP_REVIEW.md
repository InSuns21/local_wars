- MVP必須機能（地上5種、占領/生産/HQ勝利、AI Easy/Normal、セーブ/ロード、スカーミッシュ10マップ）は **充足**。
- ただし、将来拡張時の運用・品質面で改善余地あり（下記）。

## 今回の補正（MVP成立のために追加修正）

- HQ勝利判定の座標依存バグ修正
  - 以前: `x===0` 前提でHQ識別しており、実マップで誤判定の可能性
  - 現在: HQを左右端（最小x/最大x）で判定し、開始時誤勝利を回避

## 残リスク（MVP外だが将来の不具合要因）

- HQ判定は「左右端HQ」前提
  - 3陣営/特殊マップ/複数HQマップでは専用メタ定義が必要
- 永続化は `localStorage` ベース
  - データ量増加やリプレイ肥大化時に IndexedDB へ移行余地
- E2Eは React Testing Library による疑似E2E
  - 実ブラウザ操作（Playwright等）での最終導線保証は未導入
