# 2026-03-08-suicide-drones-and-counter-drone

自爆ドローン、対ドローン防空車、防空歩兵、ドローン戦向けマップを追加する実行計画です。

最終更新: 2026-03-08
状態: active

## 目的
- 低コストの使い捨て航空戦力として `自爆ドローン` を追加する
- ドローン偏重を抑える受け皿として `対ドローン防空車` と `防空歩兵` を追加する
- 工場からの複数ドローン生産と自動配置を、既存生産ルールを大きく壊さずに実装する
- ドローン戦が成立するマップと設定プリセットを追加する

## 確定した前提
- 自爆ドローンは `DRONE` サブカテゴリを持つ
- 自爆ドローンの `最大燃料は 50`、`ターン終了燃料消費量は 1`
- 自爆ドローンは `海上ユニット` を攻撃対象に含む
- 自爆ドローンは `空中補給機で補給不可`
- 対ドローン防空車の `最大弾数は 6`
- 防空歩兵の `moveRange は 2`、`コストは 4000`
- ドローンが有効な時の AI 生産比率上限は `50%` とし、詳細設定で変更可能にする
- ドローン向け新マップは `3件` 追加する

## Phase 0: 仕様の確定
- `UnitType`、設定キー、プリセット追加方針を固定する
- 自爆ドローンの戦闘解決順を `与ダメージ -> 被ダメージ -> 自壊` で固定する
- 迎撃の発火条件は `移動侵入時のみ` で固定する
- ドローンの工場上限管理に `originFactoryCoord` を持たせるか、別テーブルで持つかを決める
- 工場の通常生産とドローン生産は同ターン併用不可とする

## Phase 1: 型と定義の追加
- `UnitType` に `SUICIDE_DRONE`、`COUNTER_DRONE_AA`、`AIR_DEFENSE_INFANTRY` を追加する
- ドローン判定用の `DRONE` サブカテゴリまたは同等の型情報を追加する
- `UnitDefinition` に必要なら `intercept*` 系の定義を追加する
- `GameSettings` に以下を追加する
  - `enableSuicideDrones`
  - `droneInterceptionChancePercent`
  - `droneInterceptionMaxPerTurn`
- `UNIT_DEFINITIONS`、ラベル、外部SVG対応、戦闘表の定義を追加する

## Phase 2: 生産ルール拡張
- 自爆ドローンだけ `同一工場から複数生産可` にする
- 自動配置先を `工場直上 + 上右下左` の固定順で探索する
- 5マスすべて埋まっていれば生産不可にする
- 同一工場由来のアクティブドローン数を `5` までに制限する
- 同ターンの通常生産とドローン生産は併用不可にする
- UI 側に `ドローン専用の連続生産ボタン`、`n/5` 表示、生産不可理由を出す

## Phase 3: 戦闘と迎撃
- 自爆ドローンの通常攻撃、反撃、戦闘後消滅を実装する
- 戦闘解決順を `与ダメージ -> 被ダメージ -> 自壊` で固定する
- 対ドローン防空車の `1-2` 射程攻撃を実装する
- 対ドローン防空車の迎撃処理を実装する
  - 射程半径 `2`
  - 対象はドローンのみ
  - 確率判定
  - 弾数消費
  - 1ターン迎撃回数上限
- 迎撃ログと履歴表示を追加する

## Phase 4: AI と設定UI
- 設定画面にドローン有効化、迎撃設定、`AI 生産比率上限` を追加する
- プリセットに `ドローン戦` を追加する
- AI に以下の最小ヒューリスティックを入れる
  - 高価値目標が射程圏に近いとき自爆ドローン生産を検討
  - ドローンが有効なときだけ対ドローン防空車を候補に入れる
  - ドローン無効かつ空港なしマップでは防空歩兵を作らない

## Phase 5: マップとチュートリアル反映
- ドローンが強いマップを `3件` 追加する
- README / チュートリアル / 仕様索引に追記する
- 生産例外、迎撃、設定トグルを説明する

## プレイ所感 2026-03-08
`迎撃防衛線` を `ドローン戦` プリセットで実地確認した時の所感です。

### UX
- 初期の生産拠点選択が空港始まりのため、ドローンマップでも工場生産が主役だと分かりにくい
- 工場を選んで `自爆ドローン` を選ぶまで専用導線が出ないため、複数生産ルールの存在が見えづらい
- `ドローン生産` ボタンは見つければ使えるが、`現在数 n/上限` とセットで常時見せた方が理解しやすい
- 迎撃範囲が盤面から読めず、初見では「どこから先が危険か」を把握しにくい
- 経過ログが内部寄りのID表記で、プレイヤー向けの状況把握には弱い

### バランス
- 先手が `1500` コストのドローンを早期に複数展開でき、中央工場への圧が強い
- 防衛側は迎撃位置の最適解を UI 補助なしで学ぶ必要があり、攻撃側より学習コストが高い
- `迎撃防衛線` はテーマに対して、初期配置から迎撃ユニットの役割がやや伝わりにくい

### ここから確定した改善仕様
- ドローン戦向けマップでは生産UIの初期選択を工場優先にする
- 工場選択時に `ドローン枠 n/上限` と `ドローン生産` 導線を常時表示する
- ドローン自動配置候補から `HQ` を除外する
- 対ドローン防空車を選択したとき、迎撃半径を盤面表示する
- `迎撃防衛線` は中央工場か先手側工場位置を少し後ろへずらし、先手ドローンラッシュを弱める
- `迎撃防衛線` では、初期配置に対ドローン防空車を前線近くへ明示配置する

## プレイ所感 2026-03-08 滑走路争奪
`滑走路争奪` を `ドローン戦` プリセットで実地確認した時の所感です。

### UX
- マップ自体の主題は `中央空港の争奪` だが、ドローン戦プリセットでは工場生産も重要になり、主役が二重化して見える
- 実際の開始画面でも生産UIは空港始まりのため、プレイヤーはまず航空機を意識し、ドローン戦ルールの入口を見落としやすい
- 工場を選ぶとドローン生産は使えるが、空港テーマの説明と UI 導線が噛み合っておらず、ルールの理解負荷が高い

### バランス
- `滑走路争奪` は中央空港の価値が高く、ドローンよりも通常航空機の優先度が高く見えやすい
- その一方でドローンは工場から低コストで展開できるため、空港争奪と工場ドローン量産を同時に管理する必要があり、意図が分散する
- ドローン戦向けマップとして見ると、迎撃や工場周辺 5 マス運用よりも空港制圧の印象が前に出る

### ここから確定した改善仕様
- `滑走路争奪` はドローン戦向けマップではなく、通常航空戦マップとして扱う
- ドローン戦向けマップ枠は、工場運用と迎撃テーマが主役になる構成を優先する

## プレイ所感 2026-03-08 滑走路争奪 標準ルール
`滑走路争奪` を標準ルール、つまりドローンなしで実地確認した時の所感です。

### UX
- マップ説明、初期生産拠点、中央空港の見え方が一致しており、テーマの理解が早い
- 開始時に空港生産が先頭なのも自然で、プレイヤーが `まず航空ユニットを見る` 流れに違和感がない
- 中央の中立空港が盤面上で目立ちやすく、争点が読み取りやすい

### バランス
- ドローンなしでは `中央空港を早く押さえるか / 地上戦力で周辺都市を固めるか` の二択が素直に成立している
- 工場と空港の役割分担が明確で、ドローン戦プリセット時に感じた主役の競合がない
- 少なくとも初見プレイでは、通常航空戦マップとしての完成度は高く見える

### 比較メモ
- 同じ `滑走路争奪` でも、ドローン戦を重ねると `空港争奪` と `工場ドローン量産` が競合する
- 標準ルールではその競合が消え、マップ本来の意図がそのまま伝わる
- そのため `滑走路争奪` は、ドローン戦向けではなく通常航空戦向けの代表マップとして扱う方が整理しやすい

## 受け入れ条件
- 設定 OFF では自爆ドローンと対ドローン防空車は生産できない
- 設定 ON では工場から自爆ドローンを複数生産できる
- 自爆ドローンは工場の 5 スロットを超えて存在できない
- 自爆ドローンは攻撃または反撃後に消滅する
- 対ドローン防空車はドローンに強く、通常航空機には中程度、地上には弱い
- 迎撃の確率と回数上限は設定変更が反映される
- AI のドローン生産比率上限 `50%` が詳細設定から変更できる
- 防空歩兵は `moveRange 2 / cost 4000` で常時生産可能で、ドローン不要マップでは AI が避ける
- ドローン向けの新マップ `3件` が選択可能になる
