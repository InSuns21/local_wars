# 2026-03-07-support-units-and-resupply

補給車、空中補給機、補給コマンドを追加する実行計画です。

## 目的
- 燃料と弾薬の継戦管理に、施設依存だけでない運用レイヤーを追加する
- 地上戦と航空戦の補給線をユニット運用として成立させる
- 補給行動を UI / AI / テストまで一貫して接続する

## 前提
- `1 マス 1 ユニット` を維持する
- 補給車は地上ユニットのみ、空中補給機は航空ユニットのみを補給する
- 補給ユニットは攻撃、反撃できない
- 補給コマンドは `マンハッタン距離 1` の味方ユニットを対象とする
- 1 回の補給コマンドで範囲内の有効対象をまとめて補給する
- 補給量は対象ユニットの燃料、弾薬を最大値まで全回復する
- HP 回復は行わない
- 補給回数 `maxSupplyCharges` を設定化し、デフォルトは `4` とする
- 補給コマンド実行ごとに補給回数を 1 消費する
- 補給回数 0 の場合、補給コマンドは実行できない
- ターン終了時に補給拠点上なら補給回数は全回復する
  - 補給車は味方の都市・工場・HQ
  - 空中補給機は味方の空港

## 実装フェーズ

### Phase 0: 仕様反映
- `support-units-and-resupply.md` の確定仕様を参照元にする
- 補給対象カテゴリの定義を `地上 / 航空 / 海上` で整理する

### Phase 1: 型とユニット定義
- `UnitType` に `SUPPLY_TRUCK`, `AIR_TANKER` を追加する
- ユニット定義へコスト、移動、燃料、弾薬、視界を追加する
- 両ユニットを `攻撃不可` として定義する
- 補給対象カテゴリを定義に持たせる
- 補給ユニット状態に `supplyCharges` と最大値参照を持てるようにする

### Phase 2: ルールとコマンド
- `SUPPLY` コマンドを追加する
- 周囲 1 マスの味方だけを対象に取る
- 補給車は地上、空中補給機は航空だけを補給する
- 1 回の実行で範囲内の有効対象をまとめて補給する
- `supplyCharges > 0` のときだけ実行可能にする
- 対象がいない場合はコマンド失敗にする
- 実行時に `supplyCharges` を 1 消費する
- 補給後は `acted=true` にする

### Phase 3: 生産、補給、ログ
- 補給車を工場生産へ追加する
- 空中補給機を空港生産へ追加する
- 補給結果を行動ログへ出す
- 補給ユニット自身は既存施設補給ルールへ従わせる
- ターン終了時、補給車は味方の都市・工場・HQ、空中補給機は味方の空港で `supplyCharges` を全回復する

### Phase 4: UI と AI
- 補給コマンド表示と対象ハイライトを追加する
- 残補給回数と最大補給回数の表示を追加する
- 補給回数 0 のときの disabled 表示と理由表示を追加する
- 補給結果を情報パネルとログで読めるようにする
- AI に補給優先判断と退避判断を追加する
- 補給回数残量と帰投先を加味した行動判断を追加する

### Phase 5: テスト
- unit, integration, ui テストを追加する
- 燃料 / 弾薬だけ回復し HP は回復しないことを固定する
- 補給回数消費、0 での実行不可、拠点での全回復を固定する

## 受け入れ条件
- 補給車と空中補給機を生産できる
- 補給コマンドで周囲 1 マスの味方へ燃料と弾薬を補給できる
- 1 回の補給実行で範囲内の有効対象をまとめて補給できる
- 対象種別が地上 / 航空で正しく分かれる
- 補給回数のデフォルト値は `4` で、設定変更できる
- 補給回数は補給コマンド実行で 1 消費される
- 補給回数 0 の場合、補給コマンドを実行できない
- 補給車は味方の都市・工場・HQ、空中補給機は味方の空港でターン終了時に補給回数が全回復する
- 補給ユニットは攻撃も反撃もできない
- 補給 UI とログで結果が分かる
