# 2026-03-10-naval-units-and-ports

海上ユニットと港湾タイル実装の実行計画です。既に `UnitType` と基礎定義は存在するため、今回は `SEA / PORT` のルール接続、海上ユニット行動、UI、AI、マップの順で段階実装します。

最終更新: 2026-03-10
状態: active
関連仕様: [../../product-specs/naval-units-and-ports.md](../../product-specs/naval-units-and-ports.md)

## ゴール
- `SEA` と `PORT` を正式に海戦ルールへ接続する
- 海上ユニット 6 種を最低限遊べる形で実装する
- 海戦テーマの新規スカーミッシュマップ 4 種を追加する
- 既存の航空、輸送、補給、施設破壊、保存、AI と整合させる

## フェーズ

### Phase 1: PORT / SEA の基盤接続
- `SEA` の通行ルールを `NAVAL / AIR のみ` に確定する
- `COAST` を独立タイルとして追加し、陸地扱いの上陸帯として接続する
- `PORT` の生産、補給、修理、収入、破壊停止を海上ユニットへ接続する
- `PORT` を海ユニットが留まれる海かつ陸の複合タイルとして扱う
- `PORT` のツールチップと盤面表示を更新する
- `SEA` に地形防御補正がないことをルールとテストで固定する

### Phase 2: 海上ユニットの最低実装
- 駆逐艦
  - 通常移動、通常戦闘、対潜可視化
- 揚陸艦
  - cargo 2、搭載、降車、破壊時 cargo 同時消滅
  - 降車先は原則 `COAST / PORT` のみに制限する
  - ただし `PORT` 上の揚陸艦だけは、搭載中ユニットの通常進入ルールで隣接タイルへ降車可能にする
- 潜水艦
  - 海上ステルス、隣接または敵港直上のみ可視
- 戦艦
  - 長射程砲撃、施設砲撃
- 補給艦
  - 海上版 `SUPPLY`

### Phase 3: 空母と艦載機運用
- cargo 2 の艦載機保管
- `発艦 / 収容` コマンド
- `発艦 / 収容` は cargo ルール準拠で `移動後でも実行可`
- 艦載機は cargo 中に燃料消費を停止する
- 艦載機は毎ターン `燃料 / 弾薬 50% 回復`、`HP 1 回復` を行う
- `50% 回復` は最小 `1` 保証で丸める
- 空母が `PORT` 停泊中なら艦載機 HP 回復に `停泊時設定値` を使う
- 艦載中の `燃料 / 弾薬回復量` と `通常時 / 停泊時 HP 回復量` を設定項目化する
- 停泊時 HP 回復量のデフォルト値は `1` とする
- UI で艦載数と艦載中ユニットを表示

### Phase 4: AI と UX
- `PORT` があるマップだけ海上生産を検討する分岐を追加する
- 駆逐艦の対潜優先、揚陸艦の上陸判断、戦艦の沿岸砲撃判断を追加する
- 盤面上で `戦艦砲撃範囲 / 補給艦補給範囲 / 潜水艦可視状態` を分かりやすくする

### Phase 5: 海戦向けマップ 4 種
- `海 + 島`
  - 上陸戦入門
  - 想定サイズ: `16x16` から `18x18`
  - 想定拠点数: 工場 `各陣営1`, 空港 `0-1/陣営`, 港 `各陣営1 + 中立1`, 都市 `合計6-8`
  - `COAST` または `PORT` による明示的な上陸帯を複数置く
- `アイランドホッピング`
  - 島伝いに補給線と上陸線を伸ばす中継主題マップ
  - 想定サイズ: `20x20` から `22x22`
  - 想定拠点数: 工場 `各陣営1`, 空港 `0-1/陣営`, 港 `各陣営1 + 中立2-3`, 都市 `合計8-10`
  - 小島ごとに `PORT` と `COAST` を置き、段階上陸と中継港確保が勝敗に直結する構成にする
- `海 + 大陸`
  - 海上補給線と沿岸砲撃
  - 想定サイズ: `18x16` から `20x18`
  - 想定拠点数: 工場 `各陣営2`, 空港 `0-1/陣営`, 港 `各陣営1 + 中立1-2`, 都市 `合計8-10`
  - 大陸沿岸に `COAST` を点在させ、上陸地点の選択肢を作る
- `海 + 陸 + 空`
  - 港湾、空港、地上工場の複合戦
  - 想定サイズ: `20x18` から `22x20`
  - 想定拠点数: 工場 `各陣営2`, 空港 `各陣営1 + 中立1`, 港 `各陣営1 + 中立1`, 都市 `合計10-12`
  - 空母の艦載回復を使い分けられるよう、前線港と後方港の価値差を作る
- `空母機動戦`
  - 空母の前進運用と海上制空を主題にした海空マップ
  - 想定サイズ: `22x20` から `24x20`
  - 想定拠点数: 工場 `各陣営1`, 空港 `各陣営0-1`, 港 `各陣営2 + 中立1`, 都市 `合計8-10`
  - 前線空港に依存しすぎず、空母の発艦 / 収容 / 艦載回復が勝敗へ直結する構成にする
- `海 + 陸 + 空 (ドローン前提)`
  - 港湾、空港、工場、対ドローン防空の複合戦
  - 想定サイズ: `22x18` から `24x20`
  - 想定拠点数: 工場 `各陣営2-3`, 空港 `各陣営1`, 港 `各陣営1 + 中立1`, 都市 `合計10-14`
  - `COAST` による沿岸侵攻と港湾・空港・工場の三系統運用が干渉する構成にする

### 海戦マップ設計の共通目安
- 港は最低でも `各陣営1` を確保し、可能なら中立港を `1` 入れる
  - 理由: 海上生産と補給が 1 か所だけだと詰まりやすい
- 都市は海戦主体でも `合計6以上`、複合戦では `10前後` を目安にする
- 空港を含む場合は `16x16` 未満にしない
  - 理由: 航空と海上の両方を入れると移動導線が窮屈になる
- 海だけを広げすぎず、島、中立港、沿岸都市で中継点を作る
  - 理由: 海上ユニットの移動ターンが伸びすぎるのを避けるため
- 上陸を発生させたい地点には必ず `COAST` または `PORT` を置く
  - 理由: 揚陸艦の降車先を明示し、ルールとマップ体験を一致させるため
- `CITY / FACTORY / AIRPORT / HQ` は上下左右 4 マスすべてが `SEA` になる配置を避ける
  - 理由: 主要施設が孤立した海上構造物のように見える配置を防ぐため

## 実装順の推奨
1. `facilities / movement / terrainDefense` に `SEA / PORT / NAVAL` の正式ルールを入れる
2. 駆逐艦、潜水艦、戦艦から先に戦闘を通す
3. 揚陸艦と補給艦で既存 `cargo / SUPPLY` を再利用する
4. 最後に空母を入れる
  - 理由: 艦載機運用は cargo と航空ルールの両方に触るため依存が重い

## 注意点
- 空母は `1 マス 1 ユニット` 制約と航空機運用が衝突しやすい
  - `発艦 / 収容` を cargo ベースで扱い、盤面同居を作らない
- `COAST` は新地形名として導入するため、既存マップと移動コスト定義への影響範囲が広い
- 艦載機 `50% 回復` は最小 `1` 保証で固定する
  - 保存互換と UI 表示に影響するため
- 潜水艦は既存ステルスと整合を取る
  - `敵港直上` と `敵隣接` だけ可視に寄せる
- 戦艦は `attackRangeMax=5` のため、予測 UI と盤面マーカーが重くなりやすい
- 補給艦は `maxSupplyCharges` を既存補給仕様と共通参照する
  - 設定項目を増やしすぎない

## テスト方針
- unit
  - `SEA / PORT` 進入可否
  - `PORT` 補給、回復、生産
  - 潜水艦ステルス
  - 駆逐艦の対潜可視
  - 戦艦砲撃
  - 補給艦補給
- integration
  - `PORT` で海上ユニット生産
  - 揚陸艦の搭載、移動、降車
  - 空母の発艦、収容
  - 港湾破壊で機能停止
- ui
  - 港湾生産 UI
  - 戦艦砲撃範囲表示
  - 補給艦補給範囲表示
  - 潜水艦不可視表示
  - 揚陸艦 / 空母の cargo 表示

## 実装後に確認したい観点
- `SEA` マップで AI が海上ユニットを過剰生産しないか
- 潜水艦ステルスが FoW ON/OFF で破綻しないか
- 上陸戦マップで先手だけが極端に有利にならないか
- `海 + 陸 + 空 + ドローン` マップで UI が過密にならないか
