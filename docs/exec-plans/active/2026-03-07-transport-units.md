# 2026-03-07-transport-units

輸送車、輸送ヘリ、搭載と降車コマンドを追加する実行計画です。

## 目的
- 歩兵と軽車両の展開速度を上げる戦術レイヤーを追加する
- 航空輸送による局地展開と、地上輸送による前線再配置を成立させる
- 輸送中燃料停止と cargo 消滅まで含めて、輸送をゲームルールとして一貫させる

## 前提
- `1 マス 1 ユニット` を維持する
- 輸送ユニットは `輸送車` と `輸送ヘリ` の 2 種とする
- 輸送ユニットは攻撃、反撃できない
- 輸送車の防御力は偵察車レベル、輸送ヘリは対空に対して最弱レベルとする
- 輸送車は地上ユニットを 2 ユニットまで搭載できる
- 輸送ヘリは歩兵のみを 1 ユニット搭載できる
- 輸送ヘリは `移動後に降車可` とする
- 輸送ユニットは `搭載/降車後も移動可` とする
- 輸送ユニットは `移動後でも搭載可 / 降車可` とする
- `搭載` は 1 ターンに 1 回、`降車` も 1 ターンに 1 回までとする
- 降車可否は `地形と占有だけで判定` する
- 降車先が敵 ZOC に隣接していても許可する
- 輸送中ユニットの燃料は減らない
- 輸送ユニット破壊時、cargo もまとめて破壊される

## 実装フェーズ

### Phase 0: 仕様反映
- `transport-units.md` を参照元にする
- 搭載対象、降車条件、cargo データモデルを確定する

### Phase 1: 型とユニット定義
- `UnitType` に `TRANSPORT_TRUCK`, `TRANSPORT_HELI` を追加する
- `UnitState` に `cargo` を持てるようにする
- ユニット定義へコスト、移動、視界、搭載容量、搭載対象カテゴリを追加する
- 攻撃不可、反撃不可の定義を追加する

### Phase 2: ルールとコマンド
- `LOAD` と `UNLOAD` コマンドを追加する
- 輸送車と輸送ヘリの搭載対象判定を実装する
- 搭載時に cargo へ移し、盤面から対象ユニットを除外する
- 降車時に隣接空きタイルへ復帰させる
- 輸送ヘリの移動後降車を許可する
- 輸送ユニットの搭載後移動、降車後移動、移動後搭載を許可する
- 搭載/降車のターン内使用回数制限とターン開始時リセットを実装する
- 輸送中ユニットの燃料消費停止を turnEngine に入れる
- 輸送ユニット破壊時に cargo も削除する

### Phase 3: 生産、補給、AI
- 工場生産へ輸送車を追加する
- 空港生産へ輸送ヘリを追加する
- cargo を含むセーブ、ロード互換を入れる
- AI の輸送対象選定と搭載、降車行動を追加する
- 初期ヒューリスティックとして、占領予定の都市・工場まで歩兵が 3 ターン以上かかる場合、未搭乗の歩兵がいれば輸送ユニット生産を検討する
- 追加ヒューリスティックとして、前線再配置、空港間移送、危険地帯の歩兵退避も後続検討に入れる

### Phase 4: UI
- 搭載可能時の `搭載` コマンドを追加する
- 降車可能時の `降車` コマンドを追加する
- 輸送ユニット情報に `搭載数 / 容量` を表示する
- cargo の中身をユニット情報とログで読めるようにする
- 降車候補タイルを盤面ハイライトする

### Phase 5: テスト
- unit, integration, ui テストを追加する
- 搭載容量、搭載対象、燃料停止、破壊連動を固定する
- セーブ、ロードで cargo が壊れないことを固定する

## 受け入れ条件
- 輸送車を工場、輸送ヘリを空港で生産できる
- 輸送ユニットは攻撃、反撃できない
- 輸送車は地上ユニットを 2 つまで、輸送ヘリは歩兵を 1 つだけ搭載できる
- 搭載中ユニットは盤面に表示されず、燃料も減らない
- 搭載後も降車後も輸送ユニットは移動できる
- 輸送ユニットは移動後でも搭載、降車できる
- 搭載は 1 ターンに 1 回、降車も 1 ターンに 1 回までに制限される
- 輸送ユニット破壊時、cargo も破壊される
- UI 上で輸送状態が分かる
- AI とセーブデータが輸送状態を正しく扱える
