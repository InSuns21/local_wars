# 2026-03-06-air-units-and-airport-expansion

航空ユニット、空港、ステルス、施設爆撃を追加する実行計画です。

## 目的
- 既存の地上戦中心ルールへ航空戦を接続する
- 空港、燃料、可視ルール、施設破壊を一貫したドメインモデルで扱う
- UI と AI を含めて「遊べる状態」まで落とし切る

## 前提
- 既存コードには `AIRPORT` 地形、`FIGHTER` / `BOMBER`、燃料/弾薬、拠点耐久の基盤がある
- ただし生産は工場固定、補給は都市/工場/HQ 固定、可視ルールはステルス未対応
- 今回も `1 マス 1 ユニット` を維持し、空地同居は導入しない
- 航空ユニットは燃料 0 になった手番の終了時に消滅する
- ステルスは `味方所有施設の直上` または `味方ユニット隣接` のときだけ可視とする
- 爆撃対象は都市、工場、空港、港湾であり、HQ は対象外とする
- 施設破壊ごとの再占領コスト増加率は設定値 `n%` を使い、デフォルトは `50` とする
- 攻撃機は爆撃機の廉価版で、対戦車、対対戦車、対自走砲に特効寄り、対空車と戦闘機に特に脆弱とする
- 爆撃機は高価で高火力、高耐久の重打撃ユニットとして扱う

## 実装フェーズ

### Phase 0: 仕様確定
- `air-units-and-airport.md` の確定仕様を反映する
- 施設耐久と再占領コスト補正のデータモデルを確定する
- 攻撃機と爆撃機のステータス差を数値へ落とす

### Phase 1: 型とデータモデル拡張
- `UnitType` に `ATTACKER`, `STEALTH_BOMBER` を追加する
- ユニット定義へ航空ユニット 4 種のコスト、移動、射程、燃料、弾薬を追加する
- 攻撃機は爆撃機より低コスト、低耐久、低火力で設定する
- 攻撃機のダメージテーブルは戦車、対戦車、自走砲へ厚く、対空車と戦闘機には薄く設定する
- `TileState` に施設運用状態を持たせる
  - 例: `operational`, `bombDamage`, `recaptureCostRate`
- 空港を `capturable / supply / production` の観点で正式に扱う
- 施設ごとの `再占領コスト増加率` を保持できるようにする
- 破壊済み施設の表示用ラベルを追加する

### Phase 2: 生産と補給ルール
- `PRODUCE_UNIT` を施設種別対応に拡張する
  - 地上は工場
  - 航空は空港
- ターン開始補給を施設種別とユニット種別で分岐する
  - 航空ユニットは自軍所有空港のみ
  - 地上ユニットの既存挙動は維持
- 空港を歩兵占領可能にする
- `1 マス 1 ユニット` 前提のまま生産不可条件を維持する

### Phase 3: 燃料切れと航空行動ルール
- 航空ユニットの燃料消費タイミングを既存移動/行動ルールに接続する
- 手番終了時に `fuel === 0` の航空ユニットを消滅させる処理を `commandApplier` / `turnEngine` / `aiTurn` へ反映する
- ログと UI に消滅理由を出す
- 消滅時の勝敗判定とセーブ復元整合も確認する

### Phase 4: ステルス可視ルール
- `visibility` へステルス判定を追加する
- FoW 無効時でもステルス例外を優先する
- `味方所有施設の直上` と `味方ユニット隣接` を専用条件としてルール関数へ切り出す
- 中立施設直上では可視化しない
- 盤面描画とツールチップの可視状態を同期する

### Phase 5: 爆撃と施設破壊
- 爆撃コマンドが施設を対象に取れるよう戦闘ルールを拡張する
- 都市、工場、空港、港湾の施設耐久を爆撃で減らせるようにする
- HQ は施設爆撃の対象外にする
- 耐久 0 で `利用不可` 状態へ遷移させる
- 利用不可中は収入、補給、生産を止める
- 破壊のたびに再占領コストを `n%` 累積増加させる
  - 増加量は各回ごとに整数切り捨て
  - `n` は設定から変更可能

### Phase 6: 設定と UI 導線
- 設定画面へ `施設破壊時の再占領コスト増加率` を追加する
- 盤面で空港、港湾、機能停止施設を明示する
- 生産 UI を施設ごとに切り替える
- 航空ユニットの燃料切れ、ステルス、施設破壊状態を盤面と情報パネルで読めるようにする
- 攻撃機と爆撃機の役割差をユニット情報とツールチップで判別できるようにする
- チュートリアルと設定説明に航空ルールを追記する

### Phase 7: AI とテスト
- AI の生産評価に空港と航空ユニットを追加する
- ステルス目標の扱いと対空優先度を見直す
- 攻撃機は装甲目標優先、爆撃機は施設と高価値目標優先になるよう評価を分ける
- 施設爆撃対象の評価を追加する
- 単体、統合、UI テストを追加する

## 受け入れ条件
- 戦闘機、爆撃機、攻撃機、ステルス爆撃機を生産して対局で操作できる
- 航空ユニットは自軍空港でのみ修理、燃料回復、弾薬回復が発生する
- 航空ユニットは燃料 0 になった手番の終了時に消滅する
- ステルスユニットは `味方所有施設の直上` または `味方ユニット隣接` の時だけ敵に見える
- 攻撃機は戦車、対戦車、自走砲に対して爆撃機より費用対効果が高い
- 攻撃機は対空車と戦闘機に対して爆撃機より脆い
- 都市、工場、空港、港湾は爆撃で機能停止し、HQ は爆撃で破壊されない
- 破壊済み施設は再占領コストが設定値に基づいて累積増加する
- セーブ/ロード後も新状態が崩れない
