# air-units-and-airport

航空ユニットと空港の追加仕様です。現行実装には `AIRPORT` 地形と `FIGHTER` / `BOMBER` の基盤が一部あるため、それを正式仕様へ引き上げます。

最終更新: 2026-03-07
状態: 実装済み

## 対象スコープ
- 航空ユニット追加
  - 戦闘機
  - 爆撃機
  - 攻撃機
  - ステルス爆撃機
- 空港の追加と占領可能化
- 航空ユニットの補給・修理ルール
- 燃料切れによる航空ユニット消滅
- ステルスユニットの特殊可視ルール
- 都市、工場、空港、港湾への爆撃と施設破壊

## 確定仕様

### 航空ユニット
- 航空ユニットは `AIR` 移動タイプを使う
- 航空ユニットは地上ユニットとは別の生産ラインとする
- 航空ユニットは空港でのみ生産、回復、弾薬回復、燃料回復を行う
- 航空ユニットは移動しなくても、手番終了時に燃料を 1 消費する
- ステルス機は移動しなくても、手番終了時に燃料を 2 消費する
- 移動や攻撃で消費した燃料に加えて、上記の待機消費も同じ手番終了時に適用する
- 航空ユニットは燃料が 0 になった手番の終了時に消滅する
- 空地同居は導入しない。既存どおり `1 マス 1 ユニット` を維持する

### 空港
- 空港は歩兵で占領できる
- 空港は工場と同じ占領耐久 20 を使う
- 空港は航空ユニットの補給拠点である
- 空港を自軍が保有している場合のみ、そこで航空ユニットの回復と補給が発生する
- 中立空港、敵空港では航空ユニットの回復と補給は発生しない

### ステルス
- ステルスユニットは FoW の有無に関係なく、通常状態では敵から不可視
- 敵のステルス機は、味方所有施設の直上にいる場合だけ可視になる
- 敵のステルス機は、味方ユニットに隣接している場合だけ可視になる
- 中立施設の直上では可視にならない
- 上記条件をどちらも満たさない場合は不可視のままとする

### 爆撃による施設破壊
- 爆撃対象は都市、工場、空港、港湾とする
- HQ は爆撃で破壊できない
- 爆撃で施設耐久を 0 にすると、その施設は機能停止状態になる
- 機能停止した施設は収入、補給、生産を提供しない
- 破壊済み施設はタイル種別を保持しつつ、`operational: false` のような状態で利用不可を表す
- 施設破壊のたびに再占領コストは `n%` ずつ増える
- `n` のデフォルト値は `50` とし、設定で変更可能にする
- 再占領コスト増加は破壊のたびに累積し、増加量は各回ごとに整数切り捨てで計算する

### 攻撃機と爆撃機の役割差
- 攻撃機は爆撃機の廉価版とする
- 攻撃機は対戦車、対対戦車、対自走砲に対して特に有効とする
- 攻撃機の攻撃力と耐久力は、爆撃機より控えめにする
- 攻撃機は対空車と戦闘機に対して特に脆弱とする
- 爆撃機は攻撃機より高価で、高火力、高耐久の重打撃ユニットとする
- 爆撃機は施設破壊と高価値地上目標への打撃を主目的とする

## 補完方針

### 生産と補給の対象施設
- 航空ユニットの生産、修理、燃料回復、弾薬回復は `自軍所有の空港` のみ
- 地上ユニットの既存補給ルールは維持する

### 施設破壊の UI 表示
- 盤面ラベルとツールチップに `破壊` または `機能停止` を常設表示する
- 収入が入らない理由、生産できない理由を盤面から判断できるようにする
- 再占領コスト増加率も情報パネルかツールチップで確認できるようにする

### データモデルの基本方針
- 施設の占領耐久と爆撃耐久は、同一値を流用するか別管理にするかを実装で選べるようにする
- ただし UI 上は `現在耐久`、`機能状態`、`再占領コスト補正` を区別して扱える構造にする
