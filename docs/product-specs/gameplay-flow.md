# gameplay-flow

ゲーム開始から対局、保存、再開までの体験フローと、未完了の仕様課題を整理します。旧 `TODO_GAME_FLOW.md` の内容を、仕様と残課題に分離して再編しました。

## 現在の基本フロー
1. タイトルから `はじめから` または `つづきから` を選ぶ
2. 新規開始時は `map-select` でマップを選ぶ
3. `settings` で基本設定を確認し、必要なら詳細設定を調整する
4. `battle` で対局を進める
5. 終了時は保存付き終了、保存しない終了、勝敗確定からタイトル復帰のいずれかへ進む

## 設定とゲームロジックの接続方針
- 索敵あり/なしは視界ルールへ反映する
- 燃料補給、弾薬補給の ON/OFF は消費と回復のルールに反映する
- 収入設定はターン開始時の資金加算へ反映する
- 対局開始前に決めた設定は、保存とロードをまたいでも崩れないよう保持する

## 保存と再開の UX 方針
- つづきからは最新保存状況をタイトル上で把握できる
- セーブ選択では各スロットの可否理由をカード内表示する
- 対局終了時は既存スロット、空きスロット、上書き選択を分岐して扱う

## 将来の拡張フロー
- 実在マップや高難度 AI を増やしても、開始導線は `title -> map-select -> settings -> battle` を維持する
- 空港、港、航空、海上ユニットを追加する場合でも、設定とゲームロジックの接続点は `GameSettings` に集約する
- キャンペーンやリプレイを追加する場合は、タイトル画面の主要導線と競合しないよう別導線で増設する

## 残課題
- 実ブラウザ操作ベースの導線検証は未導入
- 保存スロット上書きフローや削除モーダルの仕様変更時は、画面フロー文書も追随させる
- 将来の複数 HQ や特殊勝利条件では、開始前説明と勝利条件表示の再設計が必要
