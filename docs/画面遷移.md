# 画面遷移設計（現行実装準拠）

最終更新: 2026-03-05
対象実装: `src/app/App.tsx` / `src/screens/*`

## 1. 画面一覧（`App.tsx` の `screen` state）

`screen` は以下の 8 状態で管理される。

- `title`（タイトル画面）
- `map-select`（マップ選択画面）
- `settings`（設定画面）
- `save-select`（セーブ選択画面）
- `credits`（クレジット画面）
- `tutorial`（チュートリアル画面）
- `audio-settings`（音量設定画面）
- `battle`（対局画面）

## 2. 全体遷移

```text
title
 ├─ はじめから -> map-select -> settings -> battle
 ├─ つづきから -> save-select -> battle（ロード成功時）
 ├─ クレジット -> credits -> title
 ├─ チュートリアル -> tutorial -> title
 └─ 音量設定 -> audio-settings -> title

battle
 ├─ その他 > ヘルプ > チュートリアル -> tutorial -> battle
 ├─ その他 > ゲーム終了 > 保存して終了 -> title（保存処理あり）
 ├─ その他 > ゲーム終了 > 保存しないで終了 -> 確認モーダル -> title
 └─ 勝敗確定時: 対局結果モーダル「タイトルへ戻る」-> title
```

## 3. 画面ごとの遷移仕様

### 3.1 タイトル画面（`title`）

- `はじめから` -> `map-select`
- `つづきから` -> `save-select`（遷移前にセーブ一覧を再読込）
- `クレジット` -> `credits`
- `チュートリアル` -> `tutorial`
  - このとき `tutorialReturnScreen = 'title'`
- `音量設定` -> `audio-settings`

### 3.2 マップ選択画面（`map-select`）

- `このマップで確定` -> `settings`（選択マップIDを保持）
- `戻る` -> `title`

### 3.3 設定画面（`settings`）

- `この設定で開始` -> `battle`
  - 新規 `GameState` を生成し `activeStore` を作成
- `戻る` -> `map-select`

### 3.4 セーブ選択画面（`save-select`）

- `このスロットで開始` -> `battle`（対象スロットにセーブがある場合）
- セーブなしスロットで開始した場合: 画面遷移せず `notice` を表示
- `削除` -> セーブ削除確認モーダル（確定で削除、画面は `save-select` のまま）
- `戻る` -> `title`

### 3.5 クレジット画面（`credits`）

- `戻る` -> `title`

### 3.6 チュートリアル画面（`tutorial`）

- `戻る` -> `tutorialReturnScreen`
  - タイトルから入った場合は `title`
  - 対局中から入った場合は `battle`

### 3.7 音量設定画面（`audio-settings`）

- BGM音量スライダーで 0〜100% を設定
- 設定値は `localStorage`（`local_wars_bgm_volume_v1`）へ即時保存
- `タイトルへ戻る` -> `title`

### 3.8 対局画面（`battle`）

- `その他 > ゲーム終了 > 保存して終了`
  - 既存紐付けスロットがあれば上書き保存して `title`
  - 紐付けなしで空きスロットがあれば自動保存して `title`
  - 空きがない場合は「保存スロット上書き選択」モーダルを表示
- `その他 > ゲーム終了 > 保存しないで終了`
  - 確認モーダル「保存しないで終了」表示
  - 確定で `title`
- `その他 > ヘルプ > チュートリアル`
  - `tutorialReturnScreen = 'battle'` にして `tutorial`
- 勝敗確定時（`winner !== null`）
  - 対局結果は別画面遷移ではなく `Dialog` モーダル表示
  - `タイトルへ戻る` で `title`

## 4. 画面遷移ではないが重要なモーダル

`screen` は変えずに重なるUI（App配下）

- セーブ削除確認モーダル（`deleteConfirmSlotId`）
- 保存しない終了確認モーダル（`showExitWithoutSaveConfirm`）
- 保存スロット上書き選択モーダル（`overwriteState`）
- 対局結果モーダル（`BattleScreen` 内 `Dialog`）

## 5. 実装との差分メモ

旧資料にあった以下は「独立画面」ではなく現行では別扱い。

- 勝敗結果画面: `battle` 内モーダル
- セーブ選択画面: 実装あり（`save-select`）
- マップ選択/設定/クレジット/チュートリアル/音量設定: 実装あり
