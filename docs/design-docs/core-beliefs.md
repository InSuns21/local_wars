# core-beliefs

LOCAL WARS の設計判断を揃えるための中核原則です。

## 原則
- ルールは UI から独立して `src/core/` に置く
- 画面導線の都合でルールを分岐させない
- 初見ユーザーの理解コストを下げる UI を優先する
- FoW、地形、占領などの重要ルールは視覚とテストの両方で保証する
- マップ差はデータで表現し、画面ロジックへ埋め込まない

## 実務上の解釈
- 新しいルール追加時は unit test を先に置く
- UI 改善でも、ルール表現に関わるなら対応テストを加える
- 画面単位の TODO は設計案と実行計画に分けて管理する
