# ui-ux-review

UI/UX の判断記録と残課題を集約します。旧 `TODO_UIUX_REVIEW.md` のうち、正式な設計判断として残すべき内容を整理したものです。

## 実装済みの主要改善
- 盤面近傍の操作性改善に向けた整理を開始した
  - 盤面ズームとスクロールナビゲーションを追加済み
  - 盤面凡例と視覚ルールの常設は未着手
- 設定画面を `基本設定` と `詳細設定` に分割し、プリセットと数値レンジ補助を追加済み
- セーブ選択の操作可否と失敗理由をカード内へ寄せた
- タイトル画面に現在状態と推奨導線を追加した
- マップ選択に比較用の詳細プレビューを追加した
- チュートリアルを `3分で分かる基本操作` と `詳細ルール` に分割した

## 継続中の設計テーマ

### 盤面体験
- 盤面近傍に文脈メニューを寄せ、左カラム依存を下げる
- 盤面凡例を常設し、色・線・バッジの意味を明文化する
- フォグ、地形、防御補正、所有者表示の視覚ルールをトークン化する

### 画面導線
- 破壊的操作は確認モーダル内でも視覚的に区別する
- 画面上部のグローバル通知は減らし、文脈近傍へ寄せる
- 初回プレイ導線と再開導線の優先度を画面単位で維持する

### 入力補助
- 数値入力は許容範囲、標準値、推奨レンジを併記する
- 設定変更でゲームバランスが崩れやすい箇所は、無制限入力よりガードを優先する

## 未完了項目

### 高優先度
- 盤面近傍の文脈メニュー
  - 対象: `BattleScreen`, `GameCanvas`
  - 狙い: 移動・攻撃・占領の主要操作を盤面近くで完結させる
- 盤面凡例と視覚トークン化
  - 対象: `GameCanvas`, `BattleScreen`
  - 狙い: 初見で盤面記号の意味を理解できるようにする

### 中優先度
- 音量設定画面の役割整理と文言改善
- クレジット画面の情報密度と可読性の整理
- アプリ全体の視覚アイデンティティ整理
- `ConfirmModal` の破壊的操作表現の差別化

### 低優先度
- モバイル viewport 前提の余白、ボタン密度、スクロール量の再点検
- 実機ベースの UI レビュー手順整備

## 実装時の受け入れ観点
- 主要操作は 1 画面内で迷わず完了できること
- エラーや不可理由は、対象カードや対象操作の近くで説明されること
- 初見ユーザー向け導線と、継続プレイヤー向け導線の優先度が衝突しないこと
- 盤面上の色や枠線が、凡例なしでもある程度推測でき、凡例で確証を持てること
